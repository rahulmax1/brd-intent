## Design Context

### Users
Mixed audience — internal product/dev team and external client stakeholders collaborating on business requirements. They're reviewing a structured intent model (actors, entities, journeys, rules) to reach consensus before development begins. Mindset is analytical and detail-oriented, but they need clarity not complexity. The AI chat panel helps them iterate on the model without leaving the review context.

### Brand Personality
**Smart, clean, modern** — like Linear. Professional without being corporate. Technical without being intimidating. The tool should feel like a well-crafted instrument that gets out of the way.

### Aesthetic Direction
- **Visual tone**: Minimal, precise, warm neutrals with cool blue accents
- **Reference**: Linear — clean, fast, keyboard-first, purposeful use of color
- **Anti-references**: Jira (dense/overwhelming), generic Bootstrap (no personality), overly playful/gamified (not taken seriously), dark/hacker aesthetic (intimidating)
- **Theme**: Light mode primary. Warm off-white (#F8F8F7) backgrounds, ACFS navy (#002C61) brand accent, blue (#0081F2) interactive accent
- **Font**: DM Sans Variable — modern geometric sans-serif

### Design Principles
1. **Clarity over decoration** — every element earns its place. Status colors, not ornamental borders. Hierarchy through weight and spacing, not visual noise.
2. **Context without clutter** — show what matters for the current task. Collapsible sections, scoped editing, progressive disclosure. Never dump everything at once.
3. **Warm precision** — professional but not cold. Warm neutrals, generous whitespace, soft shadows. The interface should feel calm and trustworthy.
4. **AI as collaborator, not magic** — the chat panel is a working tool, not a novelty. Two-step flow: AI explains what it plans to change, user confirms, then diff is generated. Transparency builds trust.
5. **Consistent rhythm** — uniform spacing scale, predictable interactions, 200ms transitions. The interface should feel like one cohesive system, not a patchwork of components.

### Typography

**Font family**: DM Sans Variable (imported from @fontsource-variable/dm-sans)
**Weights**: 400 (regular), 500 (medium), 600 (semibold), 700 (bold)

**Base sizes:**
- UI elements: 14px
- Body prose: 15px
- Small text: 13px (metadata, code)
- Tiny text: 12px (labels, badges)

**Document type scale** (doc-prose):
- **H1**: 28px, 700 weight, -0.02em tracking, 1.2 line-height
- **H2**: 22px, 600 weight, -0.01em tracking, 1.3 line-height, border-bottom
- **H3**: 17px, 600 weight
- **H4**: 15px, 600 weight
- **Body**: 15px, 400 weight, 1.7 line-height
- **Code inline**: 13px, monospace (ui-monospace), 500 weight, #F1F1EF bg
- **Code block**: 13px, monospace, 1.6 line-height

**ID display convention**: responsibility IDs (wff:r1) displayed uppercase (WFF:R1) in UI only — model data stays lowercase

### Spacing & Layout

**Spacing scale**: Tailwind 4px base unit (2, 4, 6, 8, 12, 16, 24, 32, 40, 48...)
**Border radius**:
- Standard: 8px (inputs, cards, buttons)
- Small: 6px (tooltips)
- Tiny: 4px (inline code, badges)
**Container padding**: 16px (cards), 8-12px (inputs), 4-10px (tooltips)
**Section spacing**: 40px+ vertical margins between major sections
**Generous whitespace**: Core principle — let content breathe

### Interactions

**Transitions**: 200ms standard duration for color, background, transform changes
**Hover states**:
- Nav items: rgba(0,0,0,0.04) background
- Active nav items: rgba(0,129,242,0.12) background
- Buttons: Slight opacity/background shift
**Focus states**: Blue border + 2px rgba(0,129,242,0.12) shadow
**Active states**: 1px translateY down (buttons on click)
**Scrollbars**: 4px width, #B9B9B7 thumb, hidden until container hover

### Components

**Built on ShadCN** — prefer ShadCN components over custom UI, use Base UI primitives
**Buttons**: Rounded-lg, height variants (h-6/h-7/h-8/h-9), variant system (default/outline/secondary/ghost/destructive/link)
**Badges**: text-xs, px-2, py-0.5, semantic status colors
**Cards**: White bg (#FFFFFF), subtle shadow or 1px border
**Navigation**: Active state = blue bg (--bg-blue-subtle) + blue text (--accent-blue) + 600 weight
**Tooltips**: Dark bg (#1A1A19), white text, 4-10px padding, rounded-md

### Established Patterns
- **3-column layout**: nav sidebar (200px) | model panel (left, flex) | AI chat (right, resizable 360-700px)
- **Collapsible cards**: all section items collapsed by default, expand on click. Chevron + title + status badge.
- **Entity tables**: fields as proper HTML tables (Name/Type/Description). Transitions hidden by default behind toggle.
- **Abbreviation tooltips**: VBS shipping terms (HBL, WFF, FF, DO, etc. + plurals) auto-detected with dashed underline, instant tooltip on hover.
- **Two-step AI flow**: prompt → streamed plan explanation → user confirms → JSON diff generated → approve/reject.
- **Scope tab in input**: segmented control inside the chat input box showing what scope (section/full model) the edit affects.
- **TextMate-style diff**: side-by-side diff on the /diff page with line numbers, LCS alignment, word-level highlighting. Full width, no chat panel.
- **Copy buttons**: on all diff displays — copies structured text format with +/-/~ prefixes.
- **Page loading**: blue 2px bar at top, triggers on link click, disappears on pathname change.
- **Nav hover**: subtle bg on hover (rgba(0,0,0,0.04)), stronger blue hover on active items.
- **Status labels**: "Pending Consensus" not just "Pending" — communicates what's actually happening.

## Implementation Notes

- **Global styles**: `src/app/globals.css` — all design tokens defined as CSS variables
- **Component library**: ShadCN with Base UI primitives (@base-ui/react)
- **Color usage**: Always use CSS variables (`var(--bg-page)`), never hardcoded hex values
- **Spacing**: Apply via Tailwind utility classes
- **Accessibility**: WCAG-compliant focus indicators, semantic color + text labels, keyboard navigation
- **Responsive**: Desktop-first (per project constraints), mobile not primary target
- **Dark mode**: Defined in globals.css but light mode is default and primary experience
- **Tech stack**: Next.js 16, React 19, TypeScript, Tailwind CSS 4, ShadCN, pnpm
